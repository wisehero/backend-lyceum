# 디자인 가이드

## 폰트 규칙
- 모든 한글 텍스트에는 반드시 `Pretendard` 폰트를 사용한다.
- font-family 설정 시 한글이 포함되는 요소에 `font-mono` 등 다른 폰트를 적용하지 않는다.
- `font-mono`는 영문 코드 스니펫이나 순수 숫자/영문 식별자에만 사용한다.

## 레이아웃 규칙
- 텍스트를 포함하는 요소는 콘텐츠 표현을 위한 충분한 여백(padding)을 확보한다.
- 높이를 고정(height)하지 말고 `min-height`와 `padding`을 사용하여 콘텐츠가 잘리지 않도록 한다.
- 한글은 영문보다 글자 크기가 크므로, 여유 있는 `line-height`와 `padding`을 적용한다.

## 색상 규칙
- 본문 텍스트는 배경과 충분한 명도 대비를 확보하여 가독성을 보장한다.
- 다크 모드에서 본문(secondary) 텍스트 색상은 너무 어두운 회색을 피하고, 충분히 밝은 색상을 사용한다.
- WCAG AA 기준(일반 텍스트 4.5:1, 큰 텍스트 3:1) 이상의 명도 대비를 유지한다.

## 강의 자료 형식
- 각 강의 자료는 스크롤 방식이 아닌 슬라이드 형식으로 만든다.
- 하단에 이전/다음 컨트롤 버튼을 배치하고, 키보드 방향키와 마우스 휠로도 장표를 넘길 수 있도록 한다.

## 콘텐츠 작성 규칙
- 주요 개념의 제목과 설명을 나열할 때, 대시(—)로 이어 쓰지 않고 줄바꿈(`<br>`)으로 분리하여 가독성을 높인다.

## 하지 말 것 (과거 실수 기반)

### 폰트 관련
- 한글이 포함된 요소(라벨, 상태 메시지, 버튼 텍스트 등)에 `font-mono`를 적용하지 않는다.
- CSS 클래스뿐 아니라 인라인 style에서도 한글 요소에 `font-mono`를 쓰지 않는다.
- 새 요소를 추가할 때 습관적으로 `font-mono`를 쓰지 말고, 한글 포함 여부를 먼저 확인한다.

### 바 차트 / 프로그레스 바
- 바 컨테이너에 inline `width`를 좁게 설정하여 내부 텍스트 공간을 제한하지 않는다.
- `overflow: hidden`이 텍스트를 잘라내는지 반드시 확인한다. 텍스트가 있는 요소에 `overflow: hidden`을 함부로 쓰지 않는다.
- 좁은 바에 텍스트를 넣지 말고, 바 옆이나 위에 별도 라벨로 표시한다.

### 레이아웃 / 포지셔닝
- 여러 요소를 `position: absolute`로 배치하여 겹치게 만들지 않는다. 가능하면 flexbox나 grid를 사용한다.
- 슬라이드 레이아웃에서 `align-items: center` + `overflow-y: auto` 조합을 사용하지 않는다. 콘텐츠가 뷰포트보다 클 때 위쪽이 잘린다. 대신 `align-items: flex-start` + `.slide-inner { margin: auto }`를 사용한다.
- 고정 `height`를 텍스트 포함 요소에 사용하지 않는다. 반드시 `min-height`를 사용한다.

### 슬라이드 전환
- JS에서 inline `style.opacity`나 `style.transform`을 설정하면 CSS 클래스를 override한다. 슬라이드 전환은 CSS 클래스 토글만으로 처리한다.

### 아이콘
- 이모지를 구조적 아이콘으로 사용하지 않는다. 인라인 SVG를 사용한다.
- 의미 없는 기호(`$` 등)를 장식용으로 넣지 않는다.

### 다크 모드
- `--text-secondary` 값을 `#8892a8` 같이 너무 어둡게 설정하지 않는다. `#b0b8cc` 이상의 밝기를 유지한다.
