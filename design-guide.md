# 디자인 가이드

## 폰트 규칙
- 모든 한글 텍스트에는 반드시 `Pretendard` 폰트를 사용한다.
- font-family 설정 시 한글이 포함되는 요소에 `font-mono` 등 다른 폰트를 적용하지 않는다.
- `font-mono`는 영문 코드 스니펫이나 순수 숫자/영문 식별자에만 사용한다.
- `--font-display`는 `'Outfit', 'Pretendard', sans-serif`로, 영문은 Outfit, 한글은 Pretendard로 폴백된다.

## 레이아웃 규칙
- 텍스트를 포함하는 요소는 콘텐츠 표현을 위한 충분한 여백(padding)을 확보한다.
- 높이를 고정(height)하지 말고 `min-height`와 `padding`을 사용하여 콘텐츠가 잘리지 않도록 한다.
- 한글은 영문보다 글자 크기가 크므로, 여유 있는 `line-height`와 `padding`을 적용한다.
- 본문 콘텐츠는 `max-width: 800px`로 중앙 정렬한다.

## 색상 규칙
- 본문 텍스트는 배경과 충분한 명도 대비를 확보하여 가독성을 보장한다.
- 다크/라이트 모드 모두 지원한다. CSS 변수(`:root` + `.dark`)로 관리한다.
- WCAG AA 기준(일반 텍스트 4.5:1, 큰 텍스트 3:1) 이상의 명도 대비를 유지한다.
- 장식 요소(그리드 배경, 글로우)는 읽기를 방해하지 않도록 낮은 opacity를 유지한다.

## 강의 자료 형식
- 블로그 스크롤 형식으로 작성한다 (슬라이드 아님).
- BlogLayout + TableOfContents 조합을 사용한다.
- 코드 블록은 라이트/다크 모두 어두운 배경(#1e1e2e / #060a14)을 사용한다.

## 콘텐츠 작성 규칙
- 제목과 설명 사이에 콜론(`:`) 대신 줄바꿈(`<br>`)을 사용한다.
- 표 안에 뱃지를 사용하지 않는다. 색상 텍스트로 표현한다.
- 내용을 지나치게 요약하지 않는다. 블로그 스타일에 맞게 충분히 서술한다.

## 하지 말 것 (과거 실수 기반)

### 폰트 관련
- 한글이 포함된 요소(라벨, 상태 메시지, 버튼 텍스트 등)에 `font-mono`를 적용하지 않는다.
- CSS 클래스뿐 아니라 인라인 style에서도 한글 요소에 `font-mono`를 쓰지 않는다.
- 새 요소를 추가할 때 습관적으로 `font-mono`를 쓰지 말고, 한글 포함 여부를 먼저 확인한다.

### 레이아웃 / 포지셔닝
- 여러 요소를 `position: absolute`로 배치하여 겹치게 만들지 않는다. flexbox나 grid를 사용한다.
- 고정 `height`를 텍스트 포함 요소에 사용하지 않는다. 반드시 `min-height`를 사용한다.
- `backdrop-filter`가 있는 요소 안에 `position: fixed` 오버레이를 넣지 않는다 (stacking context 문제).

### 아이콘
- 이모지를 구조적 아이콘으로 사용하지 않는다. 인라인 SVG를 사용한다.

### Astro 관련
- 코드 블록 내 `{`, `}`를 이스케이프 없이 쓰지 않는다. JSX 파싱 에러가 발생한다.
- `transition: all`을 사용하지 않는다. 변경되는 속성만 명시한다.
