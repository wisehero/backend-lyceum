# Project Rules

디자인 및 UI 관련 규칙은 [design-guide.md](./design-guide.md)를 반드시 참고한다.

## 디자인 도구
- UI/UX 디자인 및 시각화 작업 시 반드시 `/frontend-design` 스킬을 사용한다.
- 디자인 시스템(색상, 폰트, 스타일), 애니메이션, 시각 효과, 인터랙티브 요소 구현을 모두 `/frontend-design`으로 처리한다.

## 네비게이션 규칙
- 슬라이드 상단에 각 주제로 바로 이동할 수 있는 탭 네비게이션을 배치한다.
- 우측 nav dots 대신 상단 탭을 주요 네비게이션으로 사용한다.

## 슬라이드 구성 규칙
- 소주제마다 슬라이드를 1:1로 쪼개지 않는다. 상단 탭은 대주제 단위로 구성한다.
- 한 대주제가 여러 슬라이드에 걸쳐도 괜찮다. 내용이 자연스럽게 나뉘는 지점에서 분리한다.
- 반대로 대주제 전체를 한 슬라이드에 몰아넣지도 않는다. 적절한 밸런스를 유지한다.
- 슬라이드는 스크롤 가능하므로 내용이 길어도 괜찮지만, 과도하게 길면 분리를 고려한다.
